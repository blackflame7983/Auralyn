# スマート・コミットスキル (Smart Commit Skill)

ドキュメントとして機能する、構造化された読みやすいコミット履歴を作成します。

## プロセスループ

### 1. 想起と差分 (Recall & Diff)
コミットする前に、「今何をしたのか？」を自問します。
- `git diff --stat` を実行して、変更されたファイルを確認します。
- 会話の中で実行した論理的なステップを思い出します。

### 2. グルーピング
**禁止**: すべてを1つのコミットにまとめないでください。
**推奨**: 論理的な単位でグループ化します：
- グループ 1: API インターフェースの変更 (`feat: ...`)
- グループ 2: 実装 (`feat: ...`)
- グループ 3: テストの更新 (`test: ...`)

### 3. コミットメッセージのルール
- **プレフィックス**: `feat`, `fix`, `refactor`, `docs`, `test`, `chore`
- **言語**: 日本語 (Japanese)
- **フォーマット**: `<プレフィックス>: <要約>`
- **禁止事項**: "AIによってコミットされました" などの署名を入れないでください。

### 4. 実行
```bash
git add <特定のファイル>
git commit -m "feat: 機能Aを追加"
```

## アンチパターン
- ❌ `git add .` (危険すぎます。一時ファイルが含まれる可能性があります)
- ❌ すべての変更を1つの「更新」コミットにする。
- ❌ ビルドを壊す部分的なコミット。
