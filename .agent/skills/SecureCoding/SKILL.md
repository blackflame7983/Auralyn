# セキュアコーディングガイドライン (Secure Coding Guidelines)

## 入力検証とパース

- **JSON パース**: 外部入力（APIボディ、クエリパラメータ）に対する `JSON.parse()` は、必ず `try/catch` ブロックでラップしてください。
  - **理由**: 不正な JSON は同期エラーをスローし、制御されていない場合、プロセス/リクエストハンドラをクラッシュさせます。
  - **参照**: `docs/CODING_GUIDELINES.md` (詳細がある場合) を参照してください。
- **検証**: すべての外部入力（ユーザー入力、APIパラメータ、環境変数）を検証およびサニタイズしてください。

## 認可 (Authorization)

- **IAM ロール**: AWS認証には環境変数ではなく、IAMロールを使用してください。
- **検証**: 機密性の高い操作には、多層的な検証を実装してください。

## エラー処理

- **グレースフル・デグラデーション**: クリティカルでないパースが失敗した場合、500エラーをスローするのではなく、警告をログに記録してフォールバックまたはスキップしてください。
