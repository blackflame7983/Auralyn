# リサーチ・アシスタント (Research Assistant - NotebookLM Style)

深いリサーチ、要約、および複数のドキュメント間の関連性の発見にこのスキルを使用します。

## コアワークフロー

1.  **コンテキスト収集**: 関連するすべてのファイル、ドキュメントURL、またはテキストスニペットを特定します。
2.  **統合**: 発見事項を保存するための「ノートブック」（概念的または実際のアーティファクト）を作成します。
3.  **クエリ**: 統合されたデータに対して具体的な質問を投げかけます。

## リサーチ・アーティファクト

リサーチ結果を保存する Markdown アーティファクトを作成します:
- `docs/research/YYYY-MM-DD-<topic>.md`
（※VSTHostプロジェクトでは、`.gemini/antigravity/brain/` 配下も活用してください）

## 分析プロンプト

複雑なトピックを分析する場合:
- **要約**: "主な3つの論点は何か？"
- **接続**: "ファイルAはファイルBとどのように関連しているか？"
- **批判**: "何が欠けているか、または矛盾しているか？"

## スマート・ディスカバリー

ユーザーが「Xについて何を知っておくべきか？」と尋ねた場合、広範な検索を実行し、重要性の低いノイズを無視して、主要なテーマを統合します。
