# 変更履歴生成ツール (Changelog Generator)

技術的なGitコミットを、洗練されたユーザーフレンドリーなリリースノートに変換します。

## コアワークフロー

1.  **Git履歴のスキャン**: `git log --oneline --no-merges <since>..HEAD`
2.  **分類**: コミットをグループ化します（新機能 ✨、改善 🔧、修正 🐛）。
3.  **翻訳**: 技術用語をメリット重視の言葉に書き換えます。
4.  **フォーマット**: きれいなMarkdownを出力します。

## 使用ガイド

ユーザー: "先週の変更履歴を作成して"
アクション:
1.  実行: `git log --since="1 week ago" --oneline --no-merges`
2.  出力を分析。
3.  Markdownを生成。

## スタイルガイドライン

- **新機能**: "チームワークスペース" ( "feat: add workspace model" ではなく)
- **修正**: "タイムゾーンの混乱を解消" ( "fix: bug in TimeZoneUtil" ではなく)
- **除外**: `chore`, `test`, `refactor` は、重要なパフォーマンス改善でない限り非表示にします。

## 出力例

```markdown
# アップデート - 3月10日の週

## ✨ 新機能
- **チームワークスペース**: メンバーを招待してプロジェクトを共同管理できるようになりました。

## 🐛 修正
- 画像のアップロードに失敗する問題を修正しました。
```
