# ファイル整理 (File Organizer)

ファイルの種類、日付、または目的に基づいて、乱雑なディレクトリを体系的に整理します。

## コアワークフロー

1.  **分析**:
    - `ls -la` で範囲を確認します。
    - `find . -type f | ...` で種類をカウントします。
2.  **計画の提案**:
    - 「Before」と「After」の構造を示します。
    - 「削除」候補（重複など）を確認します。
3.  **実行**:
    - `mkdir -p new/folder`
    - `mv old.file new/folder/`
4.  **報告**: アクションの概要を報告します。

## 安全ルール

- **事前の確認なしに削除しないこと**: 明示的な確認がない限り、削除は禁止です。
- **移動の記録**: 何がどこに行ったかを記録します。
- **メタデータの保持**: `mv` または `cp -p` を使用します。

## 一般的なパターン

- **ダウンロード**: 拡張子でソート（`.pdf` -> Documents, `.dmg` -> Installers）。
- **プロジェクト**: 言語またはフレームワークでグループ化。
- **アーカイブ**: X日より古いアイテムを `_Archive/` に移動。
