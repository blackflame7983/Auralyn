# プロジェクト・コアルール (Project Core Rules)

これらのルールは、エージェントがこのプロジェクト (VSTHost) で作業する際に常に遵守すべき事項です。

## 1. 言語設定 (Language Settings)
- **日本語の優先**: ソースコード内のコメント、コミットメッセージ、Pull Requestの本文、およびユーザーへの説明は、原則として「日本語」で記述してください。
- **例外**: スキルファイルや内部的な自動化スクリプトなど、エージェント専用のアーティファクトについては、明確性を重視して英語を使用しても構いませんが、チーム共有のドキュメントは日本語を推奨します。

## 2. 安全性 (Safety & Security)
- **承認必須アクション**: リモートリポジトリへのプッシュ (`git push`) や、破壊的な変更を含む操作は、必ず実行前にユーザーの明示的な承認を得てください。自動実行は禁止です。
- **機密情報の保護**: 環境変数や資格情報をログに出力したり、コミットしたりしないでください。

## 3. ブランチ戦略 (Branching Strategy)
- **作業用ブランチの作成**: 作業を開始する際は、必ず目的に応じた新しいブランチを作成してください（例: `feature/`, `fix/`）。
- **直接コミットの禁止**: `master` や `main` ブランチへの直接コミットは禁止です。

## 4. 規約の遵守 (Compliance)
- **ドキュメント参照**: 実装作業を行う前に、必ず `docs/CODING_GUIDELINES.md` およびルートディレクトリの `.agent/AGENTS.md` (存在する場合) を参照し、定義されたコーディング規約、エラースタイル、ディレクトリ構造に従ってください。
- **エラーハンドリング**: 外部入力に対する処理（`JSON.parse`等）では、必ず `try/catch` ブロックを使用し、アプリケーションのクラッシュを防いでください。
