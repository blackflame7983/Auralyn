# VST Host UI/UX 徹底改善計画書 (統合版 / 2026-01-12)

## 概要
本ドキュメントは、VSTホストアプリ（vst-host）に対し、ターゲット層である「Youtube配信者」「Discordユーザー」の視点から徹底的な批判と改善提案を行うものです。
ターゲット層は**オーディオエンジニアではありません**。「ノイズを消したい」「声を良くしたい」という目的のためにツールを使用しており、設定や専門用語そのものには興味がありません。

現状は「機能は足りてきているが、親切さが足りない」状態です。DAW的な前提（Host/Buffer/SampleRate）をそのまま露出しているため、**“ノイズを消したい/声を良くしたいだけ”**の層にとって心理的負担が大きいです。

**ゴール: 5分以内に音が出て、事故らず運用できるアプリにする**

---

## P0: 今すぐ直すべき致命的な問題 (Critical)

ユーザーの信頼を損なう、または離脱の直接的な原因となる問題です。

### 1. 絵文字・アイコンの表示品質と信頼性
**問題**: ガイドやヒントで使用されている絵文字が環境によって文字化け（`??`）し、未完成・不審なソフトに見える恐れがある。また、デザイン的な一貫性を欠く。
**該当箇所**:
- `components/features/Guide/OBSGuideModal.tsx` (Stepヘッダーのアイコンなど)
- `components/features/SetupWizard/SetupWizardModal.tsx` (診断結果の ✅ 🔊 ⚠️ 🎯)
**改善策**:
- **絵文字を完全廃止し、アプリ全体のアイコンを Material Symbols (SVG) に統一する**。
- 既存の `lucide-react` も順次置き換え、アプリ全体で「信頼感のあるツール」としてのデザイン統一を図る。

### 2. 言語の壁（英語混じりの排除）
**問題**: アプリの至る所で英語と日本語が混在しており、ユーザーに「海外製の難しいソフトを使わされている」というストレスを与える。特にエラーメッセージや専門用語が英語なのは致命的。
**該当箇所**:
- **Toast**: `Unsupported file`, `Failed to load...`
- **AudioSettings**: `Host`, `Input`, `Output`, `Buffer Size`, `Sample Rate`
- **UI**: `EDIT`, `MASTER`
**改善策**:
- **完全日本語化**: ユーザーの目に入るテキストは100%日本語にする。
- **専門用語の意訳**:
    - `Buffer Size` → **「遅延調整 (バッファサイズ)」**
    - `Sample Rate` → **「音質設定 (サンプルレート)」**
    - `Input` → **「マイク入力」**
    - `Output` → **「出力先 (Discord/OBS)」**

### 3. "放送事故" 対策（パニックボタン）の欠如
**問題**: 配信中にハウリングや異音が発生した際、初心者はパニックになる。現在のUIには即座に音を止める/スルーさせる手段がない。
**改善策**:
- **「緊急バイパス（全エフェクトOFF）」スイッチ**をヘッダーの目立つ位置に常設する。
- 押下時は全エフェクトをバイパスし、**「生音」**を出力する（無音にするのではなく、会話が継続できるようにする）。

## P1: 大きな改善が必要な問題 (Major)

ユーザビリティを大きく損なっている、DAW由来の複雑さの解消。

### 4. オーディオ設定の "DAW臭さ" 解消
**問題**: 「バッファサイズ 512 samples」と表示されても、ユーザーは良し悪しを判断できない。ヒントが隠されているのも不親切。
**改善策**: **「簡易モード」と「詳細モード」の分離**。
- **簡易モード (デフォルト)**:
    - **遅延設定**: [ 高品質・低遅延 ] --- [ バランス (推奨) ] --- [ 安定性重視 ] の3段階選択（内部で128/512/1024を割り当て）。
    - **音質**: [ 配信/通話 (48kHz) ] --- [ 音楽制作 (44.1kHz) ]
- **詳細モード**: 従来の数値指定UIへの切り替えトグルを用意。

### 5. オンボーディングの "突き放し" 解消 (VB-CABLE)
**問題**: セットアップウィザードで「インストールしてください」と言われた後の導線が弱い。
**改善策**:
- 診断結果画面に **「インストールしました（再スキャン）」** という大きなアクションボタンを配置。
- 押下後にデバイスリストをリロードし、設定を自動適用するフローを組む。

### 6. OBS連携の UX 改善
**問題**: `localhost` や `4455` を手入力させるのはハードルが高い。
**改善策**:
- **初期値入力済み**: `localhost`, `4455` をプレースホルダーではなく初期値（value）として設定しておく。
- **接続失敗時のヘルプ**: 失敗時に「OBSの [ツール] > [WebSocketサーバー設定] を確認」という図解ヘルプへの導線を出す。

### 7. レベルメーターの常時表示
**問題**: `LargeLevelMeter` は画面を覆うため作業の邪魔になる。しかし配信者は「音が入力されているか」を常に見たい。
**改善策**:
- **「ドック型」または「ミニフロート型」** のメーターモードを実装し、画面の隅に邪魔にならず常駐できるようにする。

## P2: 品質・世界観の向上 (Minor)

### 8. デザインの一貫性と世界観
**問題**: 管理画面的なUIとリッチなUIが混在している。
**改善策**:
- プラグインカードに「通電中」を示すLED風インジケーターを追加するなど、ゲーミング/ストリーミングツールとしての「道具感」を演出する。
- 全体的にダーク基調でアクセントカラー（ネオン系）を効かせたテーマ調整。

### 9. 破壊的操作の保護
**問題**: プラグインの削除などが簡単に行えすぎる。
**改善策**:
- 削除時には「元に戻す（Undo）」トーストを表示するか、確認ダイアログを挟む。

    - `SetupWizardModal` に「再スキャン＆適用」フローを追加。

    - デフォルト値の適用とヘルプ導線強化。

7.  全体的なデザイン・テーマのブラッシュアップ。
